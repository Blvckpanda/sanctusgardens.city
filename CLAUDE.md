## SANCTUS GARDENS CITY — AGENT CONSTITUTION

## 2. THE TECH STACK (LOCKED)

Do not suggest, install, or import alternative libraries. The stack is final.

* **Core:** Next.js 14+ (App Router ONLY), React Server Components (RSC) by default.
* **Language:** TypeScript (Strict Mode). No `any`.
* **Styling:** Tailwind CSS v4 (using CSS `@theme` variables). `shadcn/ui` for primitives.
* **Database & Auth:** Supabase (PostgreSQL). Service-role server-side ONLY for Phase 1.
* **CMS:** Sanity Studio. Fetched via GROQ. Read-only on the frontend.
* **Mapping (2D):** Leaflet.js (Client-side rendering). *Do not use Mapbox or Cesium.*
* **Immersive (3D):** Pannellum (via CDN). *Do not use Three.js directly.*
* **Motion:** Framer Motion (Primary UI), GSAP (Secondary/Hero only).

## 3. STRICT TERMINOLOGY RULE (ZONE vs. DISTRICT)

* **The Rule:** The system term is **`Zone`**.
* **Execution:** All database tables, API routes (`/api/zones`), folder structures (`features/zones/`), and TypeScript interfaces must use `Zone`.
* **Exception:** "District" is purely a marketing term used in the frontend UI copy (e.g., `<h1>Residential District</h1>`). Never use `district` in variable names, file names, or backend logic.

## 4. SECURITY & DATA GATING

Phase 1 relies on lead capture. Do not leak gated data.

* **Server-Side Gating:** High-value data (financials, brochures, interior 3D renders) must NEVER be passed to a Client Component as a hidden prop. It must be fetched securely via an API route (`/api/opportunities/[slug]/gated-assets`) only *after* Supabase verifies the user's session/lead status.
* **Validation:** Every API route must validate incoming payloads using `Zod`.

## 5. UI & DESIGN SYSTEM INVARIANTS

* **No Hardcoded Values:** Never use raw hex codes (e.g., `#0F3D2E`) or raw Tailwind color scales (e.g., `bg-green-800`). You must strictly use the CSS variables mapped in `ui-tokens.md` (e.g., `bg-primary`, `text-gold`, `border-border`).
* **No Fixed Pixels:** Never use `px` for font sizes or layout spacing. Use `rem` and Tailwind's native `--spacing-*` scale.
* **Typography:** `Cormorant Garamond` is for Display/Headings ONLY. `Inter` is for Body/UI/Buttons ONLY.

## 6. MEDIA & ASSET PIPELINES

* **Panoramas:** Never load a raw 8K equirectangular `.jpg` directly. Pannellum must be configured to point to the multiresolution tile directories generated by `generate-panorama.py` (e.g., `/public/panoramas/{scene_id}/`).
* **Before/After Slider:** Images must be identical in pixel dimension and aspect ratio (16:9). Do not write CSS hacks to crop mismatched images; reject them.
* **Cloudinary:** All standard images (renders, portraits) must be fetched via Cloudinary using dynamic format optimization (`f_auto,q_auto`).

## 7. AI OPERATING DIRECTIVES

When receiving a prompt, you will:

1. **Locate the context:** Check `progress-tracker.md` to see what Unit you are currently building.
2. **Verify the entity:** Check `business-data-model.md` to ensure you are using the exact database schema (e.g., `total_project_cost`, `leed_edge_target`).
3. **Write the code:** Prioritize Server Components. Keep files under 300 lines. Abstract complex business logic into `/services`.
4. **Report back:** Concisely explain what was built, why it aligns with the SGC Constitution, and verify that no unauthorized dependencies were added.
