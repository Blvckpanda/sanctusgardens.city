# Sanctus Gardens City (SGC)

## Content Governance & State Management Protocol

### Purpose

This document defines the strict lifecycle states, access controls, and data validation rules for the Sanity CMS and Supabase database. It establishes how content transitions from internal planning to public rendering, ensuring the Next.js frontend only consumes structurally sound, approved data.

---

### 1. Role-Based Access Control (RBAC)

Access to the data layer is strictly partitioned to prevent unauthorized structural changes.

* **System Administrators (Engineering):** Unrestricted access. Exclusive authority to create or modify `Zone`, `InfrastructureItem`, and `Landmark` entities. Exclusive authority to manage Sanity schema deployments and Supabase migrations.
* **Marketing & Sales Leads:** Granted `Publish` access. Authorized to manage `Opportunity` entities, upload Cloudinary media, and update `ConsultationLead` status pipelines in Supabase.
* **Content Editors:** Granted `Draft` access. Authorized to author entity descriptions, update statistics, and link media. Cannot bypass the `Publish` review gate.

### 2. Opportunity Lifecycle State Machine

The `lifecycle_state` field on the `Opportunity` entity is not a generic label; it acts as a strict programmatic switch that alters frontend UI behavior.

* **`Conceptual`:**
* *Frontend Behavior:* Hidden from the public production deployment. Accessible only via the Next.js Vercel Preview URLs.
* *Requirements:* Topographical map coordinates must be defined.

* **`Active Funding`:**
* *Frontend Behavior:* Visible to the public. The `InvestmentGateModal` is enforced. Gated financial data (`total_project_cost`, projections) is accessible upon valid lead capture.
* *Requirements:* High-fidelity Cloudinary renders and approved financial data must be attached.

* **`In-Development`:**
* *Frontend Behavior:* Unlocks the `BeforeAfterSlider` component on the Opportunity detail page to demonstrate physical transformation.
* *Requirements:* A strictly formatted "After" construction image must be uploaded and paired with the original Ogbogoro "Before" marshland image.

* **`Operational`:**
* *Frontend Behavior:* Removes the `InvestmentGateModal`. Changes the primary CTA from "Speak With Our Investment Team" to "Inquire for Leasing / Utilization".

### 3. Media Validation Gates

Content Editors cannot publish an entity if the attached media violates the frontend performance or layout contracts.

* **Before/After Slider Gate:** The CMS must reject any "Before" and "After" image pair that does not share the exact same pixel dimensions and a `16:9` aspect ratio.
* **Opportunity Render Gate:** Renders exceeding `500KB` must trigger an automated Cloudinary optimization webhook before being approved for the Next.js `<Image />` component.
* **Panorama Gate:** The CMS must reject raw `.jpg` equirectangular uploads. Editors must only link the directory path of the mathematically tiled output generated by the `generate-panorama.py` pipeline.

### 4. Lead Governance (Supabase)

`ConsultationLead` data is highly sensitive and subject to strict operational handling.

* **Data Masking:** Marketing teams viewing the Supabase dashboard cannot export or bulk-download leads without elevated Administrator approval.
* **State Progression:** Sales leads must be manually transitioned through the pipeline (`Created` → `Qualified` → `Contacted` → `Converted` → `Closed`) within 48 hours of the `capture_timestamp`.
